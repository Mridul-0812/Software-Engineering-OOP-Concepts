Question 1: Structural Integrity & System Dependencies

3 Marks

Explain why favoriting structural composition over standard inheritance relationships prevents the Fragile Base Class Problem when modifying large-scale system components.

Model Answer Blueprint: Inheritance creates tight coupling because a subclass depends directly on the base class's internal implementation details. If an engineer changes the base class, it can trigger cascading failures down the entire hierarchy. Composition replaces this with a weak association, where objects are combined modularly using predefined interfaces. This isolates modifications to a single module and protects system encapsulation.
NESA/Industry Marking Rubric:
  • 3 Marks: Clearly explains how inheritance causes tight coupling and details how composition uses interfaces/modular components to eliminate cascading dependencies.
  • 2 Marks: Identifies that base class changes break child objects but provides only a general description of composition.
  • 1 Mark: Provides a basic definition of composition or inheritance without linking them to system fragility.

Question 2: Database Integrity vs Object Mapping

In data modeling, what is the architectural function of a Composite Key in an Entity-Relationship Diagram (ERD), and how does it prevent data redundancy within a junction table?

Model Answer Blueprint: A composite key combines two or more individual attributes (often separate Foreign Keys from a Many-to-Many relationship) to form a single, structurally unique Primary Key. It prevents redundancy by ensuring that the exact same combination of entities cannot be logged into the table more than once.

Question 5: System Lifecycle & Lifecycle Optimization

3 Marks

In modern 3D Game Engines, explain the technical conditions that mandate placing runtime tracking camera movements inside a LateUpdate lifecycle loop rather than a standard frame-by-frame Update loop.

Model Answer Blueprint: Player translation and physical movement are calculated inside frame updates or physics update loops. If a script updates the camera's structural look-at vectors simultaneously inside the standard Update loop, the camera might execute its coordinate tracking before the character finishes moving. This results in positional lagging or visual camera jitter. LateUpdate resolves this by running exclusively after all standard updates finish processing.
NESA/Industry Marking Rubric:
  • 3 Marks: Details the frame timing conflict (character moves first, camera calculates position), explicitly names camera jitter/lag, and defines LateUpdate's role as executing post-Update loops.

  • 1 Mark: Notes that LateUpdate handles cameras but provides no technical mechanics or explanation of frame synchronization.

Question 6: Encapsulation & Guard Clauses

3 Marks

Explain why forcing structural software systems to modify object parameters via explicit Setter Methods rather than public instance attributes preserves system integrity when processing invalid state data.

Model Answer Blueprint: Direct mutations to public instance parameters allow calling scripts to accidently inject corrupt, out-of-bounds, or malicious data (e.g., negative bank deposits). Question 7: Object Relationships & Lifecycle Dependencies

2 Marks

Contrast how the lifecycle of dependent child components is handled within a strict Composition relationship versus a weak Aggregation relationship.

Model Answer Blueprint: In a composition relationship, the parent container exerts absolute ownership over the child parts. If the parent container object is purged from engine memory, the child parts are automatically destroyed too. In an aggregation relationship, ownership is weak; child components exist as external references and survive independently of the master framework container.
NESA/Industry Marking Rubric:
  • 2 Marks: Explicitly highlights the lifecycle difference: composition means combined destruction, aggregation means structural independence.

Question 8: Physics Optimization Timelines

3 Marks

An engineer notices that a tracking camera system vibrates when trailing a vehicle object governed by rigid body physics acceleration. Identify the systemic cause of this visual flaw and propose the architectural solution.

Model Answer Blueprint: Physics calculations (Rigidbodies) update on a fixed, discrete clock timeline to maintain numeric convergence stability. Standard rendering frames (Update loop) run at dynamic, variable time intervals. If camera look-at tracking calculations happen inside the variable Update routine, the vehicle's physics engine coordinate position will mismatch, creating visible jitter. The solution is migrating the tracking mechanics into the physics-synchronized `FixedUpdate` method.

  • 3 Marks: Clearly explains the timeline mismatch between fixed physics intervals and variable frame render rates, and explicitly names `FixedUpdate` as the mandatory repair location.

Light computation systems compare normal angles against real-time directional light calculations to determine how light beams hit the surface. This data dictates whether a polygon is rendered with a highlight, a shade tint, or a shadow projection.
