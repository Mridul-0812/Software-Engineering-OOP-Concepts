Pillar 2: Inheritance & Subclass Architectural Mechanics

Comprehensive operational breakdown of Object-Oriented programming boundaries. Learn how code moves from a generalized base architecture down into specific, specialized child objects using Python syntax constructs from absolute scratch.

Core Definitions: The Inheritance Hierarchy

Inheritance is a fundamental pillar of OOP that allows a class to derive properties and behaviours from another class, establishing a "is-a" relationship (e.g., a SniperRifle is a Weapon).

  • Base Class (Superclass / Parent): The generalised class that contains common attributes and methods. It serves as a blueprint for more specific versions.
  • Derived Class (Subclass / Child): The specialised class that inherits from the Base Class. It can add unique features or modify inherited ones.

Why Use Inheritance?

  • Reusability: Write code once in the parent class; all children use it automatically.
  • Extensibility: Add new features to a child class without touching the parent class code.
  • Clarity: Organises code into a logical hierarchy, reducing redundancy.

Key Technical Mechanisms

1. Method Overriding: This occurs when a Derived Class provides a specific implementation of a method that is already defined in its Base Class.

Purpose: To allow a subclass to perform a general action (like fire()) in a way that is specific to its unique characteristics.

2. The super() Function: The super() function is a built-in Python tool used to call methods from the Parent class inside the Child class.

In Constructors (__init__): It ensures the parent’s attributes (like name or damage) are properly initialised before the child adds its own.

New to Programming? Let's Break Down the Absolute Basics

Before look at complex object patterns, let's look at the foundational code terms used in Python:

  • Variables (Attributes): These are boxes inside our class that store information/data. For example, storing an asset's name or a score value. In OOP, when a variable lives inside a class template, it is formally termed an Attribute or Instance Variable.
  • Functions (Methods): These are blocks of code that perform actions. In OOP, when a function lives inside a class blueprint, it is called a Method. It defines the object's Behaviour.
  • The self Keyword: This is a reference pointer to the *specific* object instance currently being modified in computer memory. Because a single blueprint can generate thousands of individual instances, self ensures Python updates the correct entity's internal dataset.
  • The __init__ Method (Constructor): This is a specialized initialization function that automatically executes the absolute microsecond a new object instance is constructed in memory.

class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.salary = salary # State baseline

    def get_bonus(self):
        return self.salary * 0.10 # Default calculation mapping


class Manager(Employee): # Passing Parent into Parentheses links the hierarchy
    def __init__(self, name, salary, department):
        # super() executes the Parent initialization sequence cleanly
        super().__init__(name, salary) 
        self.department = department # Unique child modification

    def get_bonus(self):
        # Overridden functional calculations
        return self.salary * 0.20

        self.set_balance(opening_deposit)

Scenario: Model an enterprise production architecture sequence. Write a parent template structural layout ServiceWorker containing a private base attribute tracking __hourly_rate, accompanied by clean getter/setter doors. Construct an inheriting subclass called Contractor that overrides a functional calculation loop calculate_payroll_payout(hours_logged), integrating a localized conditional evaluation loop to process standard metrics or overtime parameters safely.

View Canonical Solution Code 
class ServiceWorker:
    def __init__(self, standard_rate):
        self.set_hourly_rate(standard_rate)

    def get_hourly_rate(self):
        return self.__hourly_rate

    def set_hourly_rate(self, numeric_rate):
        if numeric_rate <= 0:
            raise ValueError("Operation Halt: Payment rate criteria values must be positive non-zero floats.")
        self.__hourly_rate = numeric_rate


class Contractor(ServiceWorker):
    def __init__(self, standard_rate):
        super().__init__(standard_rate)

    def calculate_payroll_payout(self, hours_logged):
        if hours_logged > 38:
            overtime_hours = hours_logged - 38
            return (38 * self.get_hourly_rate()) + (overtime_hours * self.get_hourly_rate() * 1.5)
        return hours_logged * self.get_hourly_rate()
